const初始化

int i = 42;
const int ci = i;//将i的值拷贝给了ci
int j = ci;//将ci的值拷贝给了j，只是拷贝了ci的值给j，不会影响ci
  • ci的常量特征仅仅在执行改变ci的操作时才会发生作用

默认状态下，const对象只在文件内有效

不同文件中出现同名的const变量时，等价于分别定义了独立的变量。 如果希望部分文件共享这个const对象，方法是： 一个文件定义，多个文件声明

//file.cc
extern const int bufSize = fcn();
//file_1.h
extern const int bufSize;

const的引用（常量引用）

const int ci = 1024;
const int &rl = ci;//常量 = 常量
rl = 42;           //错误：rl是给常量起别名，实质是常量，不可以被赋值
int &r2 = ci;      //错误：不可以使用变量名作为常量的别名
  • 使用整型常量作为double类型的变量的别名（类比别的类型）
double dval = 3.14
const int &ri = dval;

==>

const int temp = dval;
const int &rl = temp;
  • 不可以使用整型变量作为double类型的变量的别名（类比别的类型）
//double a = 3.14;
//int &b = a;
//错误
  • 对const的引用可能引用一个并非const的对象
int i = 42;
int &rl = i;  //变量给变量其别名，r1可以修改i的值（可以变）
const int &r2 = i;//常量给变量起别名，r2不可以修改i的值（常量不能变）
r1 = 0;         //正确
r2 = 0;         //错误

指针和const

指针常量（顶层const）必须初始化,因为初始化的是它的地址

指针类型的常量 不可以改变指针指向，可以改变指向对象的值

int a;
int *const p = &a;

常量指针（底层const）

指向常量的指针 不可以改变指向对象的值，可以改变指针指向

int a;
const int *p;

指向常量的指针常量

不可以改变指向对象的值,也不可以改变指针指向

int a;
const int *const p;

###　常量不可以　＝　变量的引用

const int ci = 42;
int &ri = ci;//错误

顶层const和底层const

指针本身是一个常量（存放的地址是一个常量）顶层const 指针指向的值是一个常量（指向的值是一个常量）底层const

int i = 0;
int *const p1 = &i; //p1地址不能改变，顶层const
const int ci = 42;  //不能改变ci的值，这是一个顶层const
const int *p2 = &ci;//p2地址可以改变，底层const
const int *const p3 = p2;//靠右的const是顶层const,靠做的是底层const
const int &r = ci;  //用于声明引用的const都是底层const

底层const需要注意：

int *p = p3;        //错误，p3包含底层const的定义，而p没有
p2 = p3;            //正确，p2和p3都是底层const
p2 = &i;            //正确，int*能转换成const int*
int &r = ci;        //错误，常量不可以 = 变量的引用
const int &r2 = i;  //正确：const

constexpr变量

作用：由编译器来验证变量的值是否是一个常量表达式

  • 必须是constexpr函数才能给constexpr类型的变量初始化
const int *p = nullptr;  //p是一个指向整型常量的指针
constexpr int *q = nullptr;//q是一个指向整数的常量指针（常量指针） 所定义的变量置为了顶层                               //const

constexpr指针既可以指向常量也可以指向一个非常量
